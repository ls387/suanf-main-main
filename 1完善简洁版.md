> ## 🟡 **可选填写的表（9张）** — 提升排课质量
>
> | 表名                         | 何时需要          | 功能说明                               |
> | ---------------------------- | ----------------- | -------------------------------------- |
> | `classroom_features`         | 课程需要特殊教室  | 定义教室特性（如：计算机机房、多媒体） |
> | `classroom_has_features`     | 课程需要特殊教室  | 关联教室与特性                         |
> | `offering_requires_features` | 课程需要特殊教室  | 指定课程需要什么特性教室               |
> | `offering_weeks`             | 周次模式为 CUSTOM | 自定义周次（如：单周、双周）           |
> | `teaching_groups`            | 需要分班教学      | 定义分班（如：英语听说分A/B班）        |
> | `teacher_blackout_times`     | 教师有禁排时间    | 教师"不可排课"的时间段（硬约束）       |
> | `teacher_preferences`        | 教师有偏好时间    | 教师偏好排课时间（软约束）             |
> | `task_relation_constraints`  | 同课程有排课关系  | 同课程不同次课的关系（如：间隔≥2天）   |
> | `schedules`                  | 输出排课结果      | 存储最终排课结果（自动生成）           |

## 📌 关键技术实现

### 1️⃣ 课表网格（P1-5核心）

```css
.timetable {
  display: grid;
  grid-template-columns: 60px repeat(7, 1fr);  /* 行号+7天 */
  grid-template-rows: 40px repeat(13, 60px);   /* 表头+13节 */
}
.course {
  grid-row: span {duration};  /* 连堂课自动占行 */
}
```

### 2️⃣ 排课异步（P0-1核心）

```
POST /api/v1/scheduling/run → 返回 task_id → WebSocket推送进度 → 结果入库
```

### 3️⃣ 拖拽冲突检测（P2-8核心）

```
用户拖拽 → 调用 /conflict-check API → 实时显示冲突 → 保存需新建version+parent_id
```
