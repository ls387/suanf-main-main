```python
def has_week_overlap(weeks1, weeks2):
    """检查两个周次集合是否有重叠"""
    if not weeks1 or not weeks2:
        return True  # 保守策略：无周次信息认为有冲突
    return len(weeks1 & weeks2) > 0  # ✅ 集合交集检测
```

## 三、问题场景分析

### 场景 1：单双周课程安排 ✅ 基本支持

**示例**：

```
课程A: 高等数学I (SINGLE) - 单周 {3,5,7,9,11,13,15,17}
课程B: 高等数学I (DOUBLE) - 双周 {4,6,8,10,12,14,16}
```

**排课算法行为**：

- ✅ 如果教师不同、班级不同：可以安排在同一时间
- ⚠️ 如果班级相同：**可能会冲突**（班级冲突检测未考虑周次）

**实际测试结果**：

```sql
-- 周五第3-4节
智能制造2025-01班 | 高等数学I(SINGLE)  | 陈金喜
智慧交通2025-01班 | 高等数学I(DOUBLE)  | 熊学
```

✅ 正常工作（不同班级）

---

### 场景 2：连续周与单周冲突 ❌ 存在问题

**示例**：

```
课程A: 军事技能 (CONTINUOUS) - 全学期 {1-17}
课程B: 高等数学 (SINGLE) - 单周 {3,5,7,9,11,13,15,17}
班级：智能制造2025-03班
```

**实际发现的冲突**：

```sql
-- 周四第1节，智能制造2025-03班
军事技能(CONTINUOUS) | 高等数学I(SINGLE)
```

**原因分析**：

1. ✅ 遗传算法的教师冲突检测正确（两门课教师不同，周次重叠，但班级不同所以允许）
2. ❌ 班级冲突检测不完整（没有获取班级已排课程的周次信息）
3. **结果**：同一班级被安排了周次重叠的两门课

---

### 场景 3：冲突优化可能产生新冲突 🔴 严重问题

**示例**：

```
初始状态：
- 课程A (单周) 在 周一 1-2节
- 课程B (双周) 在 周一 1-2节（不同教师、不同班级）
- ✅ 无冲突

优化算法运行后：
- 优化算法认为 周一 1-2节 被占用
- 将某个冲突课程C调整到 周一 1-2节
- 课程C是全学期课程
- 🔴 产生新冲突！（与课程A和B都冲突）
```

**原因**：

- 优化算法的 `occupied_times` 只记录时间，不记录周次
- 认为 "周一 1-2 节" 被占用，实际上只有特定周次被占用

---

### 场景 4：自定义周次 ⚠️ 部分支持

**示例**：

```sql
-- offering_weeks 表
offering_id=99, weeks={1,3,5,8,10,12,15}  -- 不规则周次
```

**支持情况**：

- ✅ 数据加载：正确加载自定义周次
- ✅ 冲突分析：正确检测周次重叠
- ⚠️ 排课算法：只要 task.weeks 正确，就能正常工作
- ❌ 冲突优化：无法识别自定义周次，可能产生冲突

```python
# genetic_algorithm.py

def _has_time_conflict(
    self,
    teacher_id: str,
    class_ids: List[str],
    weekday: int,
    start_slot: int,
    slots_count: int,
    teacher_schedule: Dict,
    class_schedule: Dict,
    task_id: str = None,
    weeks: Set[int] = None,
    class_schedule_with_weeks: Dict = None,  # 新增参数
) -> bool:
    """检查时间冲突 - 完整考虑周次"""

    time_slots = {(weekday, slot) for slot in range(start_slot, start_slot + slots_count)}

    # 检查教师冲突（已考虑周次）
    if task_id and task_id in self.task_dict:
        task = self.task_dict[task_id]
        for tid in task.teachers:
            if teacher_schedule[tid] & time_slots:
                if weeks is None or task.weeks is None or len(task.weeks & weeks) > 0:
                    return True

    # 检查班级冲突（新增：考虑周次）
    if class_schedule_with_weeks:
        for class_id in class_ids:
            for time_key in time_slots:
                if time_key in class_schedule_with_weeks[class_id]:
                    # 获取该班级在该时间的所有课程的周次
                    for other_weeks in class_schedule_with_weeks[class_id][time_key]:
                        if weeks and len(weeks & other_weeks) > 0:
                            return True
    else:
        # 旧逻辑：保守检测
        for class_id in class_ids:
            if class_schedule[class_id] & time_slots:
                return True

    return False
```
