   ├─ 利用率优化(+200)
   └─ 教师偏好(+100)

最终得分 = 100 + 硬约束惩罚 + 软约束奖励
        = 100 + (-60000~-5000) + (30~400)

第14页 - 精英保留与停滞检测
【左侧：精英保留机制】

第1代种群
[80,75,72,68,65,60,...] ← 10%精英
   │
   ├─ 保留最优10%个体
   │  [80,75,72] → 下一代
   │
   └─ 新一代99个个体
      从精英和其他个体
      演变而来

优势：
✓ 防止优秀解被破坏
✓ 加速收敛
✓ 避免逆向进化

【右侧：停滞检测机制】

适应度曲线
    │     ╱──── 快速上升期
最佳 │  ╱╱
适应 │ ╱   停滞平台
度   └────────────────────
     0   50   100  150  代数

停滞计数器：
代1-50：快速提升 ✓
代51-100：无改进，计数++
计数 = 50 → 停止进化 ⏹️

结果：
从200代缩减为 ~120代
计算时间 -40%
第15页 - 适应度函数详解
【大标题：适应度函数 = 可行性 + 质量】

【左侧：硬约束得分（可行性）】

基础分值：100

惩罚表：
┌────────────────────┬────────┐
│ 冲突类型           │ 惩罚   │
├────────────────────┼────────┤
│ 班级冲突(最严重)   │ -80000 │
│ 容量不足           │ -60000 │
│ 教师冲突           │ -50000 │
│ 教室冲突           │ -50000 │
│ 周末排课           │ -10000 │
│ 黑名单限制         │ -8000  │
└────────────────────┴────────┘

最终硬约束得分：
100 + (-80000) + (-60000) + ... = 负数
→ 个体被淘汰 ❌

【右侧：软约束得分（质量）】

奖励表：
┌────────────────────┬────────┐
│ 优化目标           │ 加分   │
├────────────────────┼────────┤
│ 必修课白天         │ +400   │
│ 教室连续性         │ +300   │
│ 利用率优化         │ +200   │
│ 教师偏好满足       │ +100   │
│ 学生负荷均衡       │ +50    │
└────────────────────┴────────┘

最终软约束加分：
+400 +300 +200 +100 = +1000
→ 方案质量好 ✓

【底部总结】
总体适应度 = 100 + 硬约束惩罚 + 软约束加分
示例：100 - 5000 + 400 = -4500  (良好的可行方案)
