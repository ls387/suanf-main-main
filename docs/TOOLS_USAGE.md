# 工具详细使用手册

## 工具概览

| 工具        | 文件                              | 用途   | 主要功能                 |
| ----------- | --------------------------------- | ------ | ------------------------ |
| 📊 数据检查 | `check_data_scale.py`             | 排课前 | 检查数据规模，推荐参数   |
| 🧬 遗传算法 | `suan2.py`                        | 排课   | 执行排课，保存结果       |
| 📋 结果导出 | `view_schedule.py`                | 排课后 | 导出 Excel，查看摘要     |
| 🔍 冲突分析 | `analyze_conflicts.py`            | 排课后 | 检测冲突，交互式优化     |
| 📈 约束分析 | `show_unsatisfied_constraints.py` | 排课后 | 检测软约束违反，生成报告 |

## 1. 数据检查工具 (`check_data_scale.py`)

### 功能

在排课前检查数据规模，推荐合适的算法参数。

### 使用方法

```powershell
python check_data_scale.py
```

### 输出示例

```
================================================================================
排课数据规模检查报告
================================================================================

【数据规模】
  教学任务数：    125
  教师总数：      45
  班级总数：      32
  教室总数：      28
  周次模式：      CONTINUOUS, SINGLE, DOUBLE, CUSTOM

【预计工作量】
  估计排课槽位数：   1250
  工作日可用槽位：   2800
  覆盖率：           44.6% (~中等)

【参数推荐】
  种群大小：     200
  进化代数：     300
  预计运行时间：  45 分钟

【风险警告】
  ⚠️  班级 CS2024-01 课程过多 (18 门)
  ⚠️  教师 田老师 负荷过重 (12 门课)
  ✅ 整体数据完整性：良好
```

### 输出指标说明

| 指标     | 说明                                    |
| -------- | --------------------------------------- |
| 覆盖率   | (排课槽位数) / (可用槽位数)，建议 < 80% |
| 预计时间 | 覆盖率低下时时间短，反之时间长          |
| 风险警告 | 数据异常或资源紧张的标记                |

### 常见问题

**Q: 推荐参数不合适？**

A: 可手动调整：

- 如果覆盖率 > 80%，增加 population 和 generations
- 如果时间限制，降低这两个参数

## 2. 排课工具 (`suan2.py`)

### 功能

使用遗传算法生成排课方案，保存到数据库。

### 基本用法

```powershell
# 标准排课
python suan2.py --version 1 --population 200 --generations 300

# 快速测试
python suan2.py --version 1 --population 80 --generations 100

# 自定义参数
python suan2.py --version 2 \
  --population 250 \
  --generations 400 \
  --crossover-rate 0.7 \
  --mutation-rate 0.2 \
  --elitism-size 5 \
  --max-stagnation 50
```

### 参数详解

#### 必需参数

| 参数        | 说明        | 示例             |
| ----------- | ----------- | ---------------- |
| `--version` | 排课版本 ID | `1`, `2`, `test` |

#### 重要参数

| 参数            | 默认值 | 范围    | 说明                         |
| --------------- | ------ | ------- | ---------------------------- |
| `--population`  | 200    | 50-500  | 种群大小，越大越好但耗时越长 |
| `--generations` | 300    | 50-1000 | 进化代数，越多越好但耗时越长 |

#### 可选参数

| 参数               | 默认值 | 范围    | 说明                       |
| ------------------ | ------ | ------- | -------------------------- |
| `--crossover-rate` | 0.7    | 0.5-0.9 | 交叉率，一般 0.7           |
| `--mutation-rate`  | 0.2    | 0.1-0.5 | 变异率，一般 0.2           |
| `--elitism-size`   | 5      | 1-20    | 精英保留数                 |
| `--max-stagnation` | 50     | 10-100  | 最大停滞代数，用于提前停止 |

### 输出

**控制台输出：**

```
第 1 代：最优分: 850.5 (班级冲突: 2)
第 2 代：最优分: 892.3 (班级冲突: 1)
...
第 300 代：最优分: 956.7 (班级冲突: 0) ← 最终结果

排课完成
  成功排布：120 门课程
  冲突数：
    班级冲突：0
    教师冲突：0
    教室冲突：0
  方案已保存到数据库（版本 1）
```

**日志文件：**

- `scheduling.log`：详细日志

**数据库：**

- `schedules` 表：排课结果
- `schedule_versions` 表：版本信息

### 参数推荐方案

#### 快速测试（5-10 分钟）

```powershell
python suan2.py --version test --population 80 --generations 100
```

**适用：** 验证数据完整性、测试代码改动

#### 标准运行（30-60 分钟）

```powershell
python suan2.py --version 1 --population 200 --generations 300
```

**适用：** 正常排课场景

#### 深度优化（2+ 小时）

```powershell
python suan2.py --version 2 --population 300 --generations 500
```

**适用：** 追求高质量，时间充足

#### 渐进式优化

```powershell
# 第 1 轮：快速探索
python suan2.py --version 1 --population 100 --generations 100

# 第 2 轮：基础优化
python suan2.py --version 2 --population 200 --generations 300

# 第 3 轮：精细优化
python suan2.py --version 3 --population 250 --generations 400
```

### 故障排查

**问题：运行非常缓慢**

```powershell
# 检查参数
python check_data_scale.py

# 降低参数
python suan2.py --version test --population 50 --generations 50

# 确认无语法错误后再提高参数
```

**问题：排课结果很差（冲突多）**

```powershell
# 增加资源
python suan2.py --version 2 --population 300 --generations 500

# 检查数据
python check_data_scale.py
```

## 3. 结果导出工具 (`view_schedule.py`)

### 使用方法

```powershell
python view_schedule.py 1

python view_schedule.py 2
```

## 4. 冲突分析与优化工具 (`analyze_conflicts.py`)

### 功能

- 统计三类冲突（班级、教师、教室）
- 检测容量不足、浪费等问题
- 交互式三阶段优化

### 使用方法

```powershell
# 分析冲突
python analyze_conflicts.py 1

# 只分析不优化
python analyze_conflicts.py 1 --analyze-only
```

### 工作流程

```
1. 加载排课结果
   ↓
2. 统计冲突
   ├─ 班级冲突
   ├─ 教师冲突
   └─ 教室冲突
   ↓
3. 显示冲突统计和示例
   ↓
4. 进入优化流程
   ├─ 第一阶段：容量不足
   ├─ 第二阶段：时间冲突
   └─ 第三阶段：软约束优化
   ↓
5. 保存优化结果
```

## 5. 约束分析工具 (`show_unsatisfied_constraints.py`)

### 功能

检测 7 个维度的软约束违反，生成详细 Excel 报告。

### 使用方法

```powershell
python show_unsatisfied_constraints.py 1
```

### 检测维度

| 维度        | 检测内容                  | 可优化性      |
| ----------- | ------------------------- | ------------- |
| 1. 教师偏好 | 教师偏好/避免时间满足情况 | ✅ 可优化     |
| 2. 利用率   | 教室利用率过低/过高       | ✅ 可优化     |
| 3. 日负荷   | 班级每天课程是否过多      | ⚠️ 部分可优化 |
| 4. 任务关系 | 同课程多次上课的关系约束  | ✅ 可优化     |
| 5. 时段偏好 | 必修课晚上/周末安排       | ⚠️ 部分可优化 |
| 6. 教室连续 | 同课程是否使用多个教室    | ✅ 可优化     |
| 7. 跨校区   | 教师跨校区通勤            | ⚠️ 部分可优化 |

## 完整工作流示例

### 场景：初次排课和优化

```powershell

python show_unsatisfied_constraints.py 1
# 生成约束分析报告

# 6. 如果还有问题，深度优化
python suan2.py --version 2 --population 300 --generations 500
python analyze_conflicts.py 2
```
