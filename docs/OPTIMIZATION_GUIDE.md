# 优化功能完整指南

## 一、容量优化

### 目标和原则

**三大目标：**

1. ✅ 容量充足（硬约束）
2. ✅ 浪费可控（软约束）
3. ✅ 利用率优化（软约束）

### 优化机制

#### 容量浪费过滤

排课时自动排除浪费过大的教室：

```
班级规模    最大浪费率    示例
小班(<30)   50%          25人 → 30-50人教室 ✅
中班(30-60) 40%          45人 → 45-75人教室 ✅
大班(>60)   30%          80人 → 80-115人教室 ✅

避免:
25人 → 100人教室 (浪费75% > 50%) ❌
45人 → 120人教室 (浪费62.5% > 40%) ❌
```

#### 利用率评分

目标利用率 **80% - 95%**，最优点 **87.5%**

```python
利用率区间      评分权重   说明
80% - 95%      最优      [0, 0.15]
70% - 80%      次优      [0.10, 0.25]
95% - 100%     可接受    [0.08, 0.18]
60% - 70%      勉强      [0.20, 0.30]
< 60%          重度      [1.0+]
```

**效果预期：**

```
优化前：平均利用率 72%（浪费15个课程）
优化后：平均利用率 85%（浪费3个课程）↑ 13%
```

### 使用建议

1. **首次排课**

   ```powershell
   python suan2.py --version 1 --population 200 --generations 300
   ```

2. **查看利用率**

   ```powershell
   python view_schedule.py 1
   ```

3. **如果利用率仍然不理想**
   - 增加种群和代数重新排课
   - 或手动调整 `genetic_algorithm.py` 中的 `utilization_waste` 权重（300 → 500）

## 二、冲突分析与优化工具

### 快速流程图

```
排课完成
    ↓
python analyze_conflicts.py <version>
    ↓
显示冲突统计（班级/教师/教室）
    ↓
第一阶段：容量不足冲突优化
    ├─ 检测：教室容量 < 学生数
    ├─ 方案：查找更大的教室
    └─ 确认：(y/n)
    ↓
第二阶段：时间冲突优化
    ├─ 检测：教师/班级/教室冲突
    ├─ 方案：查找无冲突的时间
    └─ 确认：(y/n)
    ↓
第三阶段：软约束优化
    ├─ 3.1 教室利用率
    ├─ 3.2 教师偏好
    ├─ 3.3 任务关系约束
    └─ 确认：(y/n)
    ↓
✅ 优化完成
```

### 功能详解

#### 阶段 1：容量不足冲突

**自动化过程：**

```
识别容量不足课程
    ↓
查找同一时间该容量充足的教室
    ↓
选择容量最接近的教室（最小化新的浪费）
    ↓
用户确认并应用
```

**示例输出：**

```
为 高等数学 寻找更大的教室...
  当前教室: A101 (容量45)
  学生数: 50
  ✓ 找到方案: A301 (容量55, 利用率90.9%)
```

#### 阶段 2：时间冲突优化

**支持的冲突类型：**

- 班级冲突：同班级不能同时上课
- 教师冲突：同教师不能同时授课
- 教室冲突：同教室不能被多班同时占用

**优化策略：**

```
对每个冲突课程：
  1. 尝试工作日其他时间（优先）
  2. 尝试周末时间
  3. 如果无法解决，标记为无法自动优化
```

#### 阶段 3：软约束优化

##### 3.1 教室利用率优化

```powershell
找到利用率过低的课程（< 50%）
    ↓
计算目标容量范围（80%-95%利用率）
    ↓
查找范围内的可用教室
    ↓
只有提升 ≥ 10% 时才调整
```

**示例：**

```
体育课程
  当前：大操场 (200座位, 30人, 利用率15%) ⚠️
  优化后：小操场 (40座位, 30人, 利用率75%) ✅
  提升：+60% 利用率
```

##### 3.2 教师偏好优化 ⭐ 新增

```
如果教师设置了偏好时间（PREFERRED）：
  1. 检测当前课程是否在偏好时间
  2. 尝试将课程调整到偏好时间
  3. 确保不违反硬约束

如果教师设置了避免时间（AVOIDED）：
  1. 检测当前课程是否在避免时间
  2. 尝试将课程移离避免时间
  3. 优先选择工作日
```

##### 3.3 任务关系约束优化 ⭐ 新增

支持的关系约束：

| 约束类型               | 说明               | 示例                        |
| ---------------------- | ------------------ | --------------------------- |
| REQUIRE_SAME_DAY       | 两次课必须在同一天 | 高等数学I分两段，都要周一上 |
| AVOID_CONSECUTIVE_DAYS | 两次课不能连续     | 高等数学I不能周一周二连续   |
| MIN_DAYS_APART         | 最少间隔N天        | 高等数学I间隔至少2天        |

**优化过程：**

```
找到违反约束的任务对
    ↓
尝试调整第一个任务到合适时间
    ↓
如果失败，尝试调整第二个任务
    ↓
如果仍失败，标记为难以优化
```

### 实战案例

#### 场景 1：初次排课后优化

## 四、完整优化工作流

### 推荐流程

```
1️⃣ 初次排课
   python suan2.py --version 1 --population 200 --generations 300

2️⃣ 约束分析
   python show_unsatisfied_constraints.py 1
   → 查看 Excel 报告，评估问题严重程度

3️⃣ 冲突优化
   python analyze_conflicts.py 1
   → 按提示进行多阶段优化

4️⃣ 结果验证
   python view_schedule.py 1
   python show_unsatisfied_constraints.py 1
   → 对比优化前后的改进

5️⃣ 迭代优化（可选）
   python suan2.py --version 2 --population 300 --generations 400
   或手动调整特定课程
```

### 参数调优建议

**如果冲突仍然较多：**

```powershell
# 增加计算资源
python suan2.py --version 2 --population 300 --generations 500

# 或调整约束权重（编辑 genetic_algorithm.py）
"class_conflict": -100000      # 提升班级冲突惩罚
"utilization_waste": 400       # 提升利用率优化权重
```

**如果只有部分冲突：**

```powershell
# 直接使用冲突优化工具
python analyze_conflicts.py 1
```

## 五、常见问题

**Q: 冲突优化能完全解决所有冲突吗？**

A: 不一定。优化工具采用启发式算法，某些情况下可能无法找到方案。此时需要：

- 增加可用教室或教师
- 手动调整某些课程
- 检查是否存在数据错误

**Q: 优化会不会产生新的冲突？**

A: 不会。优化算法在进行任何调整时都会检查硬约束，确保不违反教师/班级/教室冲突和周次限制。

**Q: 为什么某些约束无法优化？**

A: 有些约束（如"学生日负荷"）由于整体课程安排的制约，可能难以优化。这时需要：

- 重新排课时提高相应权重
- 或者接受这个约束的轻度违反

**Q: 周次冲突会被修复吗？**

A: 是的。冲突优化工具会自动避免周次冲突。如果无法避免，会标记为"无法优化"。
