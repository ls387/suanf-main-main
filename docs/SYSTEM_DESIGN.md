# 排课系统设计文档

## 一、系统概述

这是一个基于遗传算法（GA）的智能排课系统，通过进化算法和启发式优化，生成满足硬约束、优化软约束的排课方案。

**核心特性：**

- ✅ 遗传算法+智能修复的混合策略
- ✅ 完整的硬/软约束体系
- ✅ 教室容量和利用率双重优化
- ✅ 周次、偏好、关系约束支持
- ✅ 交互式冲突优化工具

## 二、主要模块

### 数据层 (`db_connector.py`)

**职责：**

- 从数据库加载教学任务、教师、班级、教室等数据
- 填充教师列表、班级列表、周次信息
- 保存排课结果到数据库

**关键类：**

- `DataLoader`：数据加载和组装
- `ScheduleRepository`：排课结果的持久化

### 数据模型 (`data_models.py`)

**核心对象：**

```python
TeachingTask      # 教学任务（包含教师、班级、时长等）
Teacher          # 教师（包含黑名单时间、偏好等）
Class            # 班级（包含学生数等）
Classroom        # 教室（包含容量、位置、设施等）
Gene             # 单个课程安排基因（时间、教室、教师）
Individual       # 个体（多个Gene的集合）
```

### 算法层 (`genetic_algorithm.py`)

**主要流程：**

1. **初始化** → 生成初始种群
   - 按优先级排序任务
   - 为每个任务分配时间、教师、教室
   - 优先尝试偏好时间，失败后尝试其他时间

2. **评估** → 计算适应度分数
   - 硬约束检测（违反则严重惩罚）
     - 教师/班级/教室时间冲突
     - 教室容量不足
     - 必需设施缺失
     - 黑名单限制
     - 周次冲突
   - 软约束评分（负分）
     - 教师偏好/避免
     - 教室连续性
     - 容量利用率
     - 学生负荷
     - 课程时段

3. **选择** → 锦标赛选择（Tournament Selection）
   - 高适应度个体更可能被选中

4. **交叉** → 单点交叉（Single-point Crossover）
   - 随机选择交叉点，合并父代基因

5. **变异** → 多元变异+智能修复
   - 变异类型：更换教师、时间、教室
   - 智能修复：尝试自动解决冲突

6. **精英保留** → 保留最优个体
   - 防止逆向进化

7. **后处理** → 冲突修复
   - 针对残留的班级冲突进行额外修复

**适应度函数计算：**

```
基础分：100
- 硬约束惩罚：-60000 ~ -100000（违反即淘汰）
- 软约束惩罚：-10 ~ -500（累积降低得分）
```

### 工具层

#### `view_schedule.py` - 结果导出

- 生成 Excel 排课表（多个视角）
- 打印控制台摘要
- 支持按日期、教师、班级、教室查看

#### `analyze_conflicts.py` - 冲突分析与优化

- 检测三类冲突：班级、教师、教室
- **三阶段优化：**
  1. 容量不足冲突（硬约束）
  2. 时间冲突（硬约束）
  3. 软约束优化（利用率、偏好等）
- 周次感知的优化（避免产生新的周次冲突）

#### `show_unsatisfied_constraints.py` - 约束分析

- 七维度检测：
  1. 教师偏好/避免
  2. 教室利用率
  3. 学生日负荷
  4. 任务关系约束
  5. 课程时段（晚上/周末）
  6. 教室连续性
  7. 教师跨校区通勤
- 生成详细 Excel 报告

## 三、约束体系

### 硬约束（不可违反）

| 约束     | 说明                   | 惩罚分  |
| -------- | ---------------------- | ------- |
| 教师冲突 | 同一教师不能同时授课   | -70000  |
| 班级冲突 | 同一班级不能同时上课   | -100000 |
| 教室冲突 | 同一教室不能同时授课   | -80000  |
| 容量不足 | 教室容量 < 学生数      | -60000  |
| 设施缺失 | 教室缺少必需设施       | -50000  |
| 黑名单   | 教师在黑名单时间内授课 | -50000  |
| 周次冲突 | 周次重叠但不应重叠     | -50000  |

### 软约束（可优化）

| 约束     | 说明                         | 权重 |
| -------- | ---------------------------- | ---- |
| 教师偏好 | 尽量安排在教师偏好时间       | 150  |
| 教师避免 | 尽量避开教师的避免时间       | 200  |
| 容量浪费 | 教室利用率目标 80-95%        | 300  |
| 教室连续 | 同课程优先使用同一教室       | 300  |
| 日负荷   | 班级每天课程 < 8 节          | 250  |
| 时段偏好 | 必修课避免晚上/周末          | 400  |
| 任务关系 | 同课程的多次上课满足关系约束 | 300  |

## 四、容量优化

### 目标

1. ✅ **容量必须充足**：教室容量 ≥ 学生数
2. ✅ **避免过度浪费**：基于班级规模的差异化标准
3. ✅ **目标利用率**：80%-95%，最优点 87.5%

### 实施方式

**容量浪费过滤：**

```python
# 筛选教室时排除浪费过大的
max_waste_ratio = {
    小班(<30人): 50%,
    中班(30-60): 40%,
    大班(>60人): 30%
}
```

**利用率评分（0分最优）：**

```python
if 0.80 <= u <= 0.95:           # 最佳区间
    return abs(0.875 - u)
elif 0.70 <= u < 0.80:          # 次优
    return 0.10 + abs(0.75 - u)
elif 0.95 < u <= 1.0:           # 接近满员
    return 0.08 + abs(0.975 - u)
elif 0.60 <= u < 0.70:          # 有浪费
    return 0.20 + abs(0.65 - u)
else:                            # 重度浪费
    return 1.0 + abs(0.85 - u)
```

## 五、周次支持

**支持的周次模式：**

- `CONTINUOUS`：全学期（1-17周）
- `SINGLE`：单周（奇数周）
- `DOUBLE`：双周（偶数周）
- `CUSTOM`：自定义周次

**周次处理：**

- ✅ 数据加载时构建周次映射
- ✅ 冲突检测时考虑周次重叠
- ✅ 优化算法避免周次冲突
- ✅ 导出时显示周次信息

## 六、关键算法细节

### 教室选择策略

1. **容量检查**
   - 容量 ≥ 学生数
   - 浪费率 ≤ 阈值
2. **时间检查**
   - 教室在给定时间段内空闲
   - 周次无冲突

3. **特征检查**
   - 拥有所有必需设施

4. **评分排序**
   - 利用率最接近 87.5%
   - 如有多个同分，随机选择

### 冲突修复策略

**智能修复（变异时）：**

- 尝试在其他时间段分配（优先工作日）
- 优先避免班级冲突
- 保持教室的连续性

**后处理修复（进化后）：**

- 逐个扫描残留班级冲突
- 为冲突任务寻找替代时间/教室
- 最多尝试 3 次

## 七、优点

- **混合策略**：GA + 启发式 + 修复机制
- **约束完整**：硬软约束系统化设计
- **灵活优化**：支持权重调整和参数配置
- **容量感知**：利用率双重优化
- **周次感知**：避免周次冲突
- **实用工具**：完整的导出、分析、优化工具链

## 八、缺点与限制

- **算法局限**：启发式算法不保证全局最优
- **运行时间**：大规模数据下耗时较长
- **数据质量**：依赖准确的教师/班级/教室数据
- **兜底缺失**：当硬约束本身无可行解时无法处理
- **可解释性**：GA 内部演化过程难以解释

## 九、参数调优建议

| 场景     | population | generations | 说明      |
| -------- | ---------- | ----------- | --------- |
| 快速测试 | 50-80      | 50-100      | 5-10分钟  |
| 标准运行 | 150-250    | 200-400     | 30-60分钟 |
| 深度优化 | 300-500    | 500+        | 2小时+    |

**调整策略：**

- 冲突多 → 增加 population 和 generations
- 时间出现不足 → 增加权重的 `teacher_conflict`、`class_conflict`
- 利用率低 → 增加 `utilization_waste` 权重
